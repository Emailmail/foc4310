## Project Overview

STM32G431CBU6 motor control (FOC) firmware template. Cortex-M4 with FPU, 128KB Flash, 32KB RAM. Generated by STM32CubeMX with hand-written algorithm and device driver layers on top of the HAL.

## Build Commands

```bash
# Configure (from Software/)
cmake --preset Debug

# Build
cmake --build build/Debug

# Or use Ninja directly
ninja -C build/Debug
```

Toolchain: `arm-none-eabi-gcc` from ARM GNU Toolchain (`C:\arm-gnu\bin`). The toolchain file is `cmake/gcc-arm-none-eabi.cmake`.

A `compile_commands.json` symlink at the repo root provides clangd indexing. Generated per-build at `build/Debug/compile_commands.json`.

## Architecture

### Layer Stack (top to bottom)

```
Algorithm/          Hand-written control algorithms (FOC, PID)
Devices/            External device drivers (AS5047P encoder, VOFA UART protocol)
BSP/                Board support — UART abstraction with multi-instance DMA+IDLE reception
Core/               STM32CubeMX-generated HAL peripheral init (ADC, TIM, SPI, etc.)
USB_Device/         USB CDC virtual COM port
Drivers/            STM32 HAL + CMSIS (auto-generated, do not modify)
Middlewares/        ARM DSP library + ST USB Device Library
```

### Pattern: Instance-Based Driver Model

All hand-written modules follow a consistent pattern:
- `XXX_Register()` allocates and initializes an instance struct, returns pointer
- `NULL` return means registration failed
- Instance structs hold all state; no globals (except in BSP UART which maintains a static array of registered instances)

### FOC (Field Oriented Control) — `Algorithm/`

Two control modes (`FOC_ControlState`):
- `FOC_OpenLoopMode` — pure open-loop with virtual electrical angle
- `FOC_EncoderOpenLoopMode` — open-loop with real angle feedback from encoder

Two modulation strategies:
- SPWM (`FOC_OpenLoop`, `FOC_EncoderOpenLoop`) — Inverse Park → Inverse Clarke → SPWM
- SVPWM (`FOC_OpenLoopSVPWM`, `FOC_EncoderOpenLoopSVPWM`) — Inverse Park → SVPWM (7-segment, sector-based)

Coordinate transforms use ARM DSP library (`arm_cos_f32`, `arm_sin_f32`). Motor parameters are configured via `FOC_InitTypedef` (bus voltage, TIM handle, pole pairs).

### TIM6 as Control Loop Timer

`HAL_TIM_PeriodElapsedCallback` for TIM6 in `main.c` runs the FOC control loop. This is where encoder reading and FOC update are called each period.

### BSP UART (`BSP/`)

Multi-instance UART abstraction supporting up to 5 UARTs (`DEVICE_UART_CNT`). Uses DMA + IDLE line detection for efficient variable-length reception (`HAL_UARTEx_ReceiveToIdle_DMA`). Each UART instance can have a device callback attached for protocol parsing.

### AS5047P Encoder (`Devices/as5047.c`)

SPI magnetic encoder. Uses 16-bit SPI frames with parity calculation. Angle reading includes a first-order low-pass filter. Output mode is compile-time selected via `AS5047P_OUTPUT_FORMAT` (0 = radians via `arm_math.h`, 1 = degrees).

### VOFA (`Devices/vofa.c`)

Firewater protocol for real-time data visualization over UART. Sends float arrays with a frame header/tail protocol for tools like VOFA+.

## STM32CubeMX Code Regeneration

Files under `Core/` contain `USER CODE BEGIN/END` markers. When regenerating from the `.ioc` file, code between markers is preserved; code outside is overwritten. The `.ioc` file is at `Template.ioc`.

The CMake structure in `cmake/stm32cubemx/CMakeLists.txt` is generated once and not re-generated — user sources in `BSP/`, `Devices/`, and `Algorithm/` are added via the top-level `CMakeLists.txt`.

## Debugging

Ozone debugger config is at `ozone/template.jdebug` (user settings in `template.jdebug.user`). The project uses Ozone for SWD debugging via J-Link.

## Pin Definitions

Key pins defined in `Core/Inc/main.h`:
- SVPWM outputs: CH1/CH1N (PA8/PB13), CH2/CH2N (PA9/PB14), CH3/CH3N (PA10/PB15)
- SPI CS for encoder: PA15
- RS485 DE: PB12
